# 小S的求助

## 题目描述

受到张老板的刺激，小S为了坚定他在同学们心目中的地位，设置了求助站，专门解决同学们学习上的各类问题。

由于大家都知道小S是卷界王中王，所以每天找他求助的人络绎不绝，这使得小S的工作量成倍增加。

小S每天还需要完成五三和教拓的内卷任务，同时还要准备演讲激励同学们内卷，所以并没有充足的时间解决所有问题。

“要是 Nicolas 和 Julian 肯帮我就好了.....”万般无奈之下，小S只能尽自己的可能帮助那些最需要帮助的同学。

同学们的求助信号用一个由小写英文字母组成的字符串来表示，字符串中连续三个字符依次组成 sos 的情况越多（即包含子串 sos 的数目越多），代表着该同学需要解决的问题越多。

由于小S电脑的键盘受到宇宙射线的袭击无法使用，就把这个艰巨的任务交给了你。

注意字符串中包含的 sos 可以有重叠，例如 sosos 算作包含 2 个 sos。

## 输入格式

第一行一个整数 n，表示需要帮助的同学的数量。

接下来有 2*n 行，每行一个由小写英文字母组成的字符串。这 2*n 行中，第 2*(i-1) 行的字符串表示第 i 个同学的名字，第 2*i 行的字符串表示第 i 个同学的求助信号。

## 输出格式

输出共两行，第一行是最需要帮助的同学的名字。如果有多个，则按照输入的顺序将他们的名字依次输出，相邻两个名字间用空格分隔。

第二行一个整数，表示最需要帮助的同学的求救信号中包含有多少个 sos 子串。

## 样例 

### 样例输入 

```
2
YMlx
zibiinghelpsosineedhelpsoswuyuzhong
suck
yineedmorehelpsoshelpcmesossoshelpmez
```

### 样例输出 

```
suck
3
```


## 提示

n<= 100，求救者名字长度不超过 $20$，求救信号长度不超过 $200$。

看到你的程序的运行答案以后，小S大吃一惊：

“怎么还有19260817个卷王有114514个问题需要解决.....受不鸟了.......”
